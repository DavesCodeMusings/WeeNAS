Hardware and Software Credits

Most of these projects provide the foundation that WeeNAS is built upon.

Raspberry Pi Single-Board Computer

Raspberry Pi is widely available, inexpensive, and getting more powerful all the time. Even an older model like the Raspberry Pi 2B+ has plenty of power for WeeNAS in a small, home network environment.

FreeBSD Operating System

FreeBSD for ARM architecture provides a solid foundation on which to build. It's FuseFS kernel module and devd service allow various types of USB storage devices to automatically appear under the /media netword share when plugged in.

Samba Project

Samba is what allows FreeBSD and other Unix-like operating systems to share files with computers running the Windows OS. When you type \\servername\shared into a Windows explorer location bar and your files appear, that's Samba at work.

Monit

Monit is a monitoring tool that keeps watch over system services and takes action to fix them if something goes wrong. This is what allows WeeNAS to be self-healing and keeps you from seeing any nasty red X statuses. It's only when a problem can't be corrected that it will alert you.

Node.js

Node.js is JavaScript running server-side. It is the language used to build the WeeNAS API, the backend to this admin tool. It makes development faster and easier, because it's the same language for backend an frontend

Breeze Icons and InkScape

Breeze icons were originally developed for the KDE desktop. Their wide selection and easy to customize SVG format makes them perfectly suited for use in other projects like the WeeNAS admin tool. Inkscape provides the tool for editing.

FreeNAS

Actually, FreeNAS has no association with the WeeNAS project other than serving as the basis for the bad pun that inspired the name. FreeNAS is a much more serious system, with much more serious hardware requirements, and a much more serious hardware price tag.
