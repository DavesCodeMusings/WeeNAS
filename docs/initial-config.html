Initial Configuration

In this section, we'll be logging into the Raspberry Pi with PuTTY on a Windows client. We'll then run an initial set of commands that will get the Pi into a state for serving up files. Don't worry, all the commands are scripted out using a handy online configuration tool. Only minimal system admin experience is required.

If you haven't plugged in the flash drive yet, do it now. Don't plug in any other storage devices at this time.

Find the IP Address of Your Raspberry Pi

  1. Start up Angry IP Scanner.
  2. Click Tools > Fetchers, from the menu bar.
  3. Select MAC Vendor from Available Fetchers column and use the arrow to place move it to Selected Fetchers column.
  4. Run a scan of your home network's IP range.
  5. Look in the MAC Vendor column of the output for Raspberry Pi Foundation.

This is your Raspberry Pi's IP address. Remember it or write it down.

Connecting Via SecureShell (SSH)

  1. Start up PuTTY.
  2. Type your Raspberry Pi's IP address into the Host Name (or IP address) field.
  3. Click the Open button.
  4. Say yes to any messages about new host keys.
  5. Log in with the initial username/password combination of freebsd/freebsd
  6. Change to the root user with the command 'su -'. Use the default password of root

Generate the Initial Configuration Script

  1. Open the WeeNAS Install web page.
  2. Type in the required trusted username(s) for anyone who will be administering WeeNAS. Uppercase letters are okay, but accounts will be generated in lowercase.
  3. Regular users can be added for anyone using WeeNAS for file storage who will not be performing system administration.
  4. Set the timezone to a location close to you. This will keep date-time stamps consistent with client machines.
  5. Browse the remaining options if you wish to customize the setup further.

All of the commands you need to configure WeeNAS are now displayed on the green screen monitor of the installer web page. Keep this information handy or simply leave the configuration web page open for later reference.

Configuring FreeBSD

If you are familiar with Unix on the command-line, you may want to open a file on the Raspberry Pi, copy all of the commands to it, and run it as a single script. You can also copy the commands, one at a time, from the installer page and paste them into the PuTTY window.

  1. Ensure you are logged in as root by running the command 'whoami'.
  2. Run each of the commands from the install script on the Raspberry Pi via PuTTY.
  3. Pay close attention to the last command. It will display the randomly assigned passwords for the trusted users. Remember these passwords!
  4. When all of the commands have been run, you can log out of PuTTY with the command 'exit'. If you configured a static IP address, restart with 'shutdown -r now'.

Verify Access for Trusted User

  1. Use PuTTY to once again connect to the Raspberry Pi.
  2. Log in with the first trusted user and the random password displayed by the previous steps.
  3. Use the command 'su -' to switch to the root user.
  4. This is a good time to change the root password using the 'passwd' command.

Verifying Access to CIFS (Samba) Shares

  1. Open up This PC or File Explorer.
  2. In the left side list, locate This PC.
  3. Right-click on This PC and choose 'Add a network location' from the list of choices on the context menu.
