Initial Configuration

In this section, we'll be using a Windows client computer to log into the Raspberry Pi using PuTTY and run an initial set of commands to get the Pi into a state for serving up files. Don't worry, all the commands are scripted out using a handy online configuration tool. Minimal brain cells are required.

If you haven't plugged in the flash drive yet, do it now.

Find the IP Address of Your Raspberry Pi

  1. Start up Angry IP Scanner.
  2. Click Tools > Fetchers, from the menu bar.
  3. Select MAC Vendor from Available Fetchers column and use the arrow to place move it to Selected Fetchers column.
  4. Run a scan of your home network's IP range.
  5. Look in the MAC Vendor column of the output for Raspberry Pi Foundation.

This is your Raspberry Pi's IP address. Remember it or write it down.

Connecting Via SecureShell (SSH)

  1. Start up PuTTY.
  2. Type your Raspberry Pi's IP address into the Host Name (or IP address) field.
  3. Click the Open button.
  4. Say yes to any messages about new host keys.
  5. Log in with the initial username/password combination of freebsd/freebsd
  6. Type su - and use the default root password of root

Generate the Initial Configuration Script

  1. Open the WeeNAS Install web page.
  2. Type in the required trusted username(s) for anyone who will be administering WeeNAS.
  3. Regular users can be added for anyone using WeeNAS for file storage who is not a system admin.
  4. Browse the remaining options if you wish to customize the setup.

All of the commands you need to configure WeeNAS are displayed on the green screen monitor of the installer web page.

Configuring the Raspberry Pi

  1. If you are familiar with Unix on the command-line, you may want to open a file on the Raspberry Pi, copy all of the commands to it, and run it as a script.
  2. You can also copy the commands, one at a time, from the installer page and paste them into the PuTTY window.
  3. Pay close attention to the last command. It will display the randomly assigned password for the first trusted user. Remember this password!
  4. When all of the commands have been run, you can log out of PuTTY with the command 'exit'. If you changed to a static IP address, restart with 'shutdown -r now'.

Verify SSH Connectivity

  1. Use PuTTY to once again connect to the Raspberry Pi.
  2. Log in with the first trusted user and the random password displayed by the previous steps.
